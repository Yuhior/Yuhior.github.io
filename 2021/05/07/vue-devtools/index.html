你的调试工具vue-devtools还好么？

发表于 2021-05-07 | 分类于 Vue

讲讲最近的事吧！最近去了杭州，体验了什么叫人家尽枕河的生活，最近心情也不太好，比较消极，我自己能想到的排解方式可能就是更加努力的工作和学习吧，古人云：世上本无事，庸人自扰之，做好自己吧！努力生活，去追逐诗和远方。

上回书说到体验vue3.0，但是在学习vue3的时候，进行敲代码你怎么能不使用我们的调试工具vue-devtools？都说磨刀不误砍柴工，那我今天也想砍柴之前磨磨刀。但是突然发现我的刀呢？我的刀… 哇哇哇。我的刀断了啊。我能这么办？




找了网上各大元老们兢兢业业整理的答案，然而并没有什么卵用….

  1. 修改Vue.config.devtools = true;
  2. 导入Vue插件是我们选择开发版本的插件（vue.js） ，不倡导引入vue.min.js啊
  3. 开启允许访问文件网址和在无痕模式下启用两项

在我满心等待欢喜的时候，欢喜却错过了你。唉无奈啊

开启自我寻找，我想着可能是我的工具不能用了 ，卸载了重新装吧。然后在谷歌商店进行重装的时候我发现了beta 版，这是什么鬼，然后我就搜了一下，看见了么？针对vue3.0 ，我赶紧跑了一下原来的项目，发现的原来2.5的项目的刀是可以用的哈哈哈哈，原来用错刀了啊。

于是进行下载beta版的vue-devtools-beta工具，启用然后重启浏览器。我的vue-devtools 用起来了。不须要其余配置，重启浏览器，直接F12能够看到新版的vue-devtool
若是不行，说明你安装的不对，不是配置问题！记住不是配置问题，看到网上不少人在说什么配置，那个是2.0强制开启的配置。

ps: 直接在谷歌商店需要梯子，没有梯子就去github上下载一波，解压之后需要再添加进去
